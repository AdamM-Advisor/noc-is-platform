## 1. FILE TYPE AUTO-DETECTION (`backend/services/file_detector.py`)

Platform menerima **5 jenis file** dengan karakteristik berbeda. Buat service yang mendeteksi jenis file secara otomatis:

    def detect_file_type(filename: str, headers: list[str]) -> dict:
        """
        Deteksi jenis file berdasarkan nama file DAN header kolom.
    
        Return:
        {
            "file_type": "site_master" | "swfm_event" | "swfm_incident" 
                         | "swfm_realtime" | "fault_center",
            "confidence": "high" | "medium" | "low",
            "reason": "Filename contains 'SWFM_ev' + headers are Title Case",
            "header_format": "upper_case" | "title_case" | "snake_case",
            "expected_columns": 7 | 72 | 73
        }
        """

### Aturan Deteksi:

    FILE 1 — SITE MASTER:
      Filename: mengandung "Naming" atau "Site_Class" atau "site_master"
      Header: UPPER_CASE → SITE_ID, SITE_NAME, ID_REGION_NETWORK, NOP_NAME, 
              SITEAREA_TO, CLASS, FLAG_SITE
      Kolom: 7
    
    FILE 2 — SWFM EVENT (historikal):
      Filename: mengandung "SWFM_ev" atau "swfm_event"
      Header: Title Case → "Ticket Number Inap", "Severity", dll
      Kolom: 72
      Ciri khas: TIDAK ada kolom "yearmonth"
    
    FILE 3 — SWFM INCIDENT (historikal):
      Filename: mengandung "SWFM_inc" atau "swfm_incident"
      Header: Title Case
      Kolom: 72
      Ciri khas: TIDAK ada kolom "yearmonth"
    
    FILE 4 — SWFM REALTIME:
      Filename: mengandung "realtime" atau "SWFM_realtime"
      Header: snake_case → "ticket_number_inap", "severity", dll  ⚠️ BERBEDA
      Kolom: 73 (ada kolom "yearmonth" tambahan)
    
    FILE 5 — FAULT CENTER:
      Filename: mengandung "Fault_Center" atau "fault_center"
      Header: Title Case
      Kolom: 72
      Ciri khas: incident_priority berisi angka besar (43, 13, dll)
    
    FALLBACK:
      Jika filename tidak match → deteksi dari header format + jumlah kolom
      Jika masih ambiguous → return confidence="low", minta user konfirmasi

## 3. PROCESSING ENGINE — SITE MASTER (`backend/services/site_master_processor.py`)

    async def process_site_master(file_path: str) -> dict:
        """
        Proses upload file Site Master (Naming_Site_Class.xlsx).
    
        Langkah:
        1. Baca file Excel (openpyxl via DuckDB atau pandas)
        2. Normalize headers (UPPER_CASE → snake_case)
        3. Validasi: harus ada 7 kolom wajib
        4. Untuk setiap baris:
           a. Resolve hierarchy:
              - id_region_network "REGIONAL3" → cari di master_regional 
                WHERE regional_alias_site_master = 'REGIONAL3'
              - Jika ditemukan → ambil regional_id
              - Jika TIDAK → log ke orphan_log, set calc_regional_id = NULL
    
              - nop_name "NOP KENDARI" → cari di master_nop
                WHERE nop_alias_site_master = 'NOP KENDARI' 
                OR nop_name = 'NOP KENDARI'
              - Jika TIDAK → log ke orphan_log
    
              - sitearea_to "TO KENDARI" → cari di master_to
                WHERE to_alias_site_master = 'TO KENDARI'
                OR to_name = 'TO KENDARI'
              - Jika TIDAK → log ke orphan_log
    
           b. Hitung enrichment (panggil enrichment_service.enrich_site):
              - Input: site_class, site_flag
              - Output: 10 kolom derived (site_category, est_technology, dll)
    
           c. UPSERT ke master_site:
              - Jika site_id sudah ada → UPDATE
              - Jika baru → INSERT
    
        5. Return: {
             total: 78675,
             inserted: 75000,
             updated: 3675,
             orphans: {regional: 0, nop: 5, to: 12},
             enriched: 78675,
             duration_sec: 45.2
           }
        """

### Strategi untuk Site Master PERTAMA KALI:

    Saat pertama kali upload, master_regional/nop/to masih KOSONG.
    Jadi semua site akan menjadi orphan.
    
    SOLUSI: Auto-populate master dari site master file.
    
    1. Kumpulkan semua unique ID_REGION_NETWORK (misal: REGIONAL0..REGIONAL12)
       → Auto-create master_regional entries:
         regional_id = "R00" s/d "R12"
         regional_alias_site_master = "REGIONAL0" s/d "REGIONAL12"
         area_id = NULL (belum tahu, user isi nanti atau dari data tiket)
    
    2. Kumpulkan semua unique NOP_NAME (61 unik)
       → Auto-create master_nop entries:
         nop_id = slugify(nop_name)  misal "NOP_KENDARI"
         nop_alias_site_master = "NOP KENDARI"
         regional_id = dari mapping REGIONAL di langkah 1
    
    3. Kumpulkan semua unique SITEAREA_TO (151 unik)
       → Auto-create master_to entries:
         to_id = slugify(sitearea_to)  misal "TO_KENDARI"
         to_alias_site_master = "TO KENDARI"
         nop_id = dari mapping NOP di langkah 2
    
    4. Tentukan regional → NOP mapping dari data:
       Groupby (ID_REGION_NETWORK, NOP_NAME) → REGIONAL3 contains NOP BOGOR
       → Set master_nop WHERE nop_alias_site_master='NOP BOGOR' 
         SET regional_id = 'R03'
    
    5. Tentukan NOP → TO mapping dari data:
       Groupby (NOP_NAME, SITEAREA_TO) → NOP BOGOR contains TO BOGOR 1
       → Set master_to WHERE to_alias_site_master='TO BOGOR 1'
         SET nop_id = 'NOP_BOGOR'
    
    Ini memastikan hierarchy ter-populate OTOMATIS dari site master.
    User tinggal assign Area nanti (dari data tiket atau manual).

        swfm_realtime, fault_center).

        1. AUTO-BACKUP database sebelum mulai
        2. Baca file (xlsx/csv/parquet) — gunakan DuckDB read_parquet atau pandas
        3. Normalize headers (Title Case → snake_case, atau langsung jika sudah snake_case)
        4. Validasi kolom minimum (site_id, severity, occured_time wajib ada)
        5. Deteksi periode: extract bulan/tahun dari occured_time atau yearmonth
        6. BATCH PROCESSING (chunk per 10.000 baris untuk efisiensi):
    
           Untuk setiap batch:
           a. HIERARCHY RESOLUTION:
              - area "AREA 4" → cari master_area WHERE area_alias = 'AREA 4'
                → calc_area_id = "AREA4"
              - regional "R08_Kalimantan" → cari master_regional

## 5. CALCULATED COLUMNS — Detail Kalkulasi

Implementasikan 17 kolom kalkulasi. Semua dihitung di Python/DuckDB SQL, BUKAN di frontend.

### 5A. Waktu Kalkulasi (4 kolom)

    # Semua dalam MENIT (minutes)
    
    # calc_response_time_min = (take_over_date - created_at) dalam menit
    # Jika salah satu NULL → NULL
    # Jika negatif → 0 (data anomali, tetap simpan tapi clamp ke 0)
    
    # calc_repair_time_min = (cleared_time - take_over_date) dalam menit
    
    # calc_restore_time_min = (cleared_time - occured_time) dalam menit
    # INI ADALAH MTTR (Mean Time To Restore)
    
    # calc_detection_time_min = (created_at - occured_time) dalam menit

### 5B. Durasi SLA (3 kolom)

    # calc_sla_duration_min:
    #   Jika holding_status = "Hold" → exclude holding time
    #   Simplified: gunakan duration_ticket dari file jika tersedia
    #   Atau: calc_restore_time_min sebagai fallback
    
    # calc_sla_target_min:
    #   Panggil sla_service.get_sla_target(site_class, site_flag, ...)
    #   Convert target (misal 90%) ke menit berdasarkan window SLA
    #   Simplified v1.0: gunakan mttr_target_min dari master_sla_target
    
    # calc_is_sla_met:
    #   True jika sla_status == "IN SLA" (dari file)
    #   ATAU jika calc_sla_duration_min <= calc_sla_target_min
    #   Prioritas: gunakan sla_status dari file jika tersedia

### 5C. Klasifikasi Waktu (7 kolom)

    # Semua dari occured_time (waktu gangguan terjadi)
    
    # calc_hour_of_day   = EXTRACT(HOUR FROM occured_time)           # 0-23
    # calc_day_of_week   = EXTRACT(DOW FROM occured_time)            # 0=Sun, 6=Sat
    # calc_week_of_month = CEIL(EXTRACT(DAY FROM occured_time) / 7)  # 1-5
    # calc_month         = EXTRACT(MONTH FROM occured_time)           # 1-12
    # calc_year          = EXTRACT(YEAR FROM occured_time)            # 2025
    # calc_year_month    = strftime(occured_time, '%Y-%m')           # "2025-07"
    # calc_year_week     = strftime(occured_time, '%Y-W%W')          # "2025-W28"

### 5D. Hierarki Resolved (4 kolom) + Source (1 kolom)

    # calc_area_id     = resolved dari area → master_area.area_alias
    # calc_regional_id = resolved dari regional → master_regional.regional_alias_ticket
    # calc_nop_id      = resolved dari nop → master_nop
    # calc_to_id       = resolved dari cluster_to → master_to
    
    # calc_source      = file_type ("swfm_event" / "swfm_incident" / 
    #                     "swfm_realtime" / "fault_center")

### Implementasi yang EFISIEN:

    JANGAN hitung per-baris di Python loop. Gunakan DuckDB SQL:
    
    1. Load data ke temporary table (temp_import)
    2. Jalankan UPDATE SQL yang menghitung semua kolom sekaligus:
    
       UPDATE temp_import SET
         calc_response_time_min = EXTRACT(EPOCH FROM (take_over_date - created_at)) / 60,
         calc_repair_time_min = EXTRACT(EPOCH FROM (cleared_time - take_over_date)) / 60,
         calc_restore_time_min = EXTRACT(EPOCH FROM (cleared_time - occured_time)) / 60,
         calc_detection_time_min = EXTRACT(EPOCH FROM (created_at - occured_time)) / 60,
         calc_hour_of_day = EXTRACT(HOUR FROM occured_time),
         calc_day_of_week = EXTRACT(DOW FROM occured_time),
         calc_week_of_month = CEIL(EXTRACT(DAY FROM occured_time) / 7.0),
         calc_month = EXTRACT(MONTH FROM occured_time),
         calc_year = EXTRACT(YEAR FROM occured_time),
         calc_year_month = strftime(occured_time, '%Y-%m'),
         calc_year_week = strftime(occured_time, '%Y-W%W'),
         calc_source = '{file_type}';
    
    3. Hierarchy resolution via JOIN:
       UPDATE temp_import t SET
         calc_area_id = a.area_id
       FROM master_area a 
       WHERE t.area = a.area_alias;
    
       (serupa untuk regional, nop, to)
    
    4. SLA resolution — batch lookup dari master_sla_target
    
    5. INSERT INTO noc_tickets SELECT * FROM temp_import 
       WHERE (ticket_number_inap, calc_source) NOT IN 
             (SELECT ticket_number_inap, calc_source FROM noc_tickets);
    
    6. DROP TABLE temp_import;
    
    Ini JAUH lebih cepat dari Python loop (detik vs menit).

## 6. DUPLICATE DETECTION

    ATURAN DEDUPLIKASI:
    
    1. Primary key logis: ticket_number_inap + calc_source
       → Tiket yang sama dari sumber berbeda = BUKAN duplikat
       → Tiket yang sama dari sumber yang sama = DUPLIKAT → skip
    
    2. Cross-source overlap:
       Satu tiket bisa muncul di multiple source (Event + Realtime).
       Ini NORMAL — kedua record disimpan, dibedakan oleh calc_source.
       Saat analisis, query akan filter by calc_source atau deduplikasi
       berdasarkan prioritas:
         fault_center > swfm_incident > swfm_event > swfm_realtime
    
    3. Re-upload bulan yang sama:
       Jika user upload file Juli 2025 dua kali →
       Tiket yang sudah ada di-skip (berdasarkan ticket_number_inap + calc_source).
       Hanya tiket BARU yang di-insert.
    
    4. Jangan gunakan DELETE + re-INSERT — terlalu berisiko.
       Gunakan INSERT ... WHERE NOT EXISTS pattern.

## 7. SUMMARY TABLE REFRESH (`backend/services/summary_service.py`)

Setelah setiap import, refresh summary tables untuk periode yang terpengaruh:

    async def refresh_summaries(year_months: list[str]):
        """
        Refresh summary_monthly dan summary_weekly untuk periode tertentu.
    
        Dipanggil setelah:
        - Upload data tiket baru
        - Perubahan master data (SLA target, hierarchy)
        - Manual refresh dari UI
    
        Langkah:
        1. Hapus summary lama untuk year_months yang terpengaruh
        2. Re-calculate dari noc_tickets
        3. Insert summary baru
        """

### Summary Monthly Calculation:

    -- Contoh: refresh summary untuk 2025-07
    -- Hitung per level hierarki (area, regional, nop, to, site)
    
    -- Level Area
    INSERT INTO summary_monthly
    SELECT
        calc_year_month as year_month,
        calc_area_id as area_id,
        NULL as regional_id, NULL as nop_id, NULL as to_id, NULL as site_id,
        NULL as severity, NULL as type_ticket, NULL as fault_level,
        COUNT(*) as total_tickets,
        SUM(CASE WHEN calc_is_sla_met THEN 1 ELSE 0 END) as total_sla_met,
        ROUND(SUM(CASE WHEN calc_is_sla_met THEN 1 ELSE 0 END) * 100.0 / 
              NULLIF(COUNT(*), 0), 2) as sla_pct,
        ROUND(AVG(calc_restore_time_min), 2) as avg_mttr_min,
        ROUND(AVG(calc_response_time_min), 2) as avg_response_min,
        SUM(CASE WHEN is_escalate = 'True' OR is_escalate = 'true' THEN 1 ELSE 0 END) as total_escalated,
        ROUND(SUM(CASE WHEN is_escalate = 'True' OR is_escalate = 'true' THEN 1 ELSE 0 END) * 100.0 / 
              NULLIF(COUNT(*), 0), 2) as escalation_pct,
        SUM(CASE WHEN is_auto_resolved = 'Auto Resolved' THEN 1 ELSE 0 END) as total_auto_resolved,
        ROUND(SUM(CASE WHEN is_auto_resolved = 'Auto Resolved' THEN 1 ELSE 0 END) * 100.0 / 
              NULLIF(COUNT(*), 0), 2) as auto_resolve_pct,
        0 as total_repeat,    -- repeat detection di Prompt selanjutnya
        0.0 as repeat_pct,
        SUM(CASE WHEN severity = 'Critical' THEN 1 ELSE 0 END) as count_critical,
        SUM(CASE WHEN severity = 'Major' THEN 1 ELSE 0 END) as count_major,
        SUM(CASE WHEN severity = 'Minor' THEN 1 ELSE 0 END) as count_minor,
        SUM(CASE WHEN severity = 'Low' THEN 1 ELSE 0 END) as count_low,
        CURRENT_TIMESTAMP as calculated_at
    FROM noc_tickets
    WHERE calc_year_month = '2025-07'
      AND calc_area_id IS NOT NULL
    GROUP BY calc_year_month, calc_area_id;
    
    -- Ulangi untuk level: regional, nop, to, site
    -- Dan untuk kombinasi severity + type_ticket (hanya level TO ke atas)

### Strategi Efisiensi:

    Level site: hanya summary "semua severity, semua tipe" (~60K/bulan)
    Level TO ke atas: tambah breakdown per severity, per type (~3.500/bulan)
    Total: ~63K baris/bulan — sangat kecil, query instant
    
    JANGAN hitung semua kombinasi severity × type × fault_level di level site.
    Ini akan menghasilkan jutaan baris yang tidak berguna.

## 8. AREA RESOLUTION DARI DATA TIKET

    MASALAH: Site master TIDAK punya kolom Area.
    Area hanya ada di data tiket ("AREA 1", "AREA 2", "AREA 4").
    
    SOLUSI: Saat upload data tiket, auto-update Area di hierarchy:
    
    1. Dari tiket, extract pasangan (regional, area):
       "R08_Kalimantan" → "AREA 4"
       "R03_Inner Jabotabek" → "AREA 2"
    
    2. Update master_regional:
       UPDATE master_regional SET area_id = 'AREA4'
       WHERE regional_alias_ticket = 'R08_Kalimantan'
       AND (area_id IS NULL OR area_id = '');
    
    3. Ini terjadi OTOMATIS saat upload tiket pertama.
       Setelah itu, area_id sudah terisi dan v_hierarchy lengkap.
    
    4. Log mapping yang ditemukan:
       "Auto-mapped R08 → AREA4 (dari 15.000 tiket)"

## CATATAN PENTING

1. **Urutan upload yang direkomendasikan**:
  
  * PERTAMA: Site Master (Naming_Site_Class.xlsx) → populate hierarchy
  * KEDUA: Data tiket (file 2-5) → bisa resolve hierarchy dari master
  * Tapi platform HARUS bisa handle urutan terbalik juga
2. **File besar (>100MB parquet)**:
  
  * Gunakan DuckDB `read_parquet()` langsung — jangan load ke pandas dulu
  * DuckDB sangat efisien membaca parquet (columnar, compressed)
3. **Concurrency**: Hanya 1 upload bisa diproses sekaligus (DuckDB single-writer)
  
  * Jika upload sedang berjalan, upload berikutnya harus menunggu
  * Tampilkan pesan: "Sedang memproses upload lain. Mohon tunggu..."
4. **Error handling**: Jika processing gagal di tengah jalan:
  
  * JANGAN commit partial data
  * Rollback → hapus temp data
  * Log error ke import_logs dengan status "failed"
  * User bisa retry
5. **Openpyxl vs DuckDB**: Untuk file .xlsx, gunakan:
  
      # Opsi 1 (recommended): pandas read_excel lalu DuckDB
      import pandas as pd
      df = pd.read_excel(file_path, engine='openpyxl')
      conn.execute("CREATE TEMP TABLE temp_import AS SELECT * FROM df")
      
      # Opsi 2: DuckDB spatial extension (jika tersedia)
      # conn.execute("INSTALL spatial; LOAD spatial;")
      # conn.execute("SELECT * FROM st_read('file.xlsx')")
  
  Untuk file .parquet:
  
      # DuckDB native — sangat cepat
      conn.execute("CREATE TEMP TABLE temp_import AS SELECT * FROM read_parquet('file.parquet')")
