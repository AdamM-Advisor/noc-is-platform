### 1D. File `start.sh`

```bash
#!/bin/bash
# start.sh — menjalankan backend + frontend bersamaan

echo "=== NOC-IS Analytics Platform ==="
echo "Author: Dr. Adam M."

# Install Python dependencies (sekali saja, di-cache oleh Replit)
if [ ! -f ".deps_installed" ]; then
    echo "[1/3] Installing Python dependencies..."
    pip install -r requirements.txt
    
    echo "[2/3] Installing frontend dependencies..."
    cd frontend && npm install && cd ..
    
    touch .deps_installed
    echo "[3/3] Dependencies installed."
fi

# Start backend (background)
echo "Starting backend on :8000..."
cd backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload &
BACKEND_PID=$!
cd ..

# Start frontend (foreground)
echo "Starting frontend on :5173..."
cd frontend
npm run dev -- --host 0.0.0.0 --port 5173

# Cleanup
kill $BACKEND_PID 2>/dev/null
```

```bash
chmod +x start.sh
```

## 4. FALLBACK PDF STRATEGY

### Jika WeasyPrint Gagal di Replit

```
WeasyPrint membutuhkan system libraries yang kompleks.
Jika replit.nix tidak cukup, gunakan strategi fallback:

OPSI A: HTML-only (tanpa PDF generation)
  → Report Card render sebagai HTML
  → User print dari browser (Ctrl+P → Save as PDF)
  → Tambahkan @media print CSS
  → INI CUKUP untuk v1

OPSI B: Ganti WeasyPrint dengan library pure-Python
  → fpdf2 (pip install fpdf2)
  → Tidak butuh system library
  → Tapi harus build PDF secara programatik (bukan HTML→PDF)
  → Lebih banyak kode, tapi pasti jalan di Replit

OPSI C: Puppeteer/Playwright (headless browser)
  → npm install puppeteer
  → Render HTML → screenshot → PDF
  → Butuh chromium (besar, ~200MB)
  → Replit paid tier saja
```

### Implementasi Opsi A (HTML Print — paling simpel):

```python
# backend/services/pdf_service.py — fallback

def generate_report_html(report_data: dict) -> str:
    """
    Generate HTML report yang print-friendly.
    User download HTML atau print dari browser.
    """
    template = jinja2_env.get_template(f"{report_data['type']}.html")
    html = template.render(**report_data)
    return html

# API:
# GET /api/reports/{id}/html → return HTML (viewable + printable)
# User buka di browser → Ctrl+P → Save as PDF
```

```css
/* Tambahkan di template HTML untuk print-friendly */
@media print {
  @page {
    size: A4;
    margin: 15mm;
  }
  body {
    font-size: 10pt;
    color: #000;
  }
  .no-print {
    display: none !important;
  }
  .page-break {
    page-break-before: always;
  }
  /* Force background colors saat print */
  * {
    -webkit-print-color-adjust: exact !important;
    print-color-adjust: exact !important;
  }
}
```

### Implementasi Opsi B (fpdf2 — pure Python):

```python
# Jika benar-benar butuh server-side PDF tanpa WeasyPrint
# pip install fpdf2

from fpdf import FPDF
import io

class ReportPDF(FPDF):
    def header(self):
        self.set_font('Helvetica', 'B', 14)
        self.cell(0, 10, 'LAPORAN NOC-IS', 0, 1, 'C')
        self.ln(5)
    
    def footer(self):
        self.set_y(-15)
        self.set_font('Helvetica', 'I', 8)
        self.cell(0, 10, f'Dr. Adam M. | NOC-IS Analytics Platform v1.0 | Hal {self.page_no()}', 
                  0, 0, 'C')

def generate_pdf_fallback(report_data: dict) -> bytes:
    pdf = ReportPDF()
    pdf.add_page()
    
    # Status banner
    pdf.set_font('Helvetica', 'B', 12)
    pdf.cell(0, 8, f"Status: {report_data['status_text']}", 0, 1)
    pdf.set_font('Helvetica', '', 10)
    pdf.multi_cell(0, 5, report_data['narrative'])
    pdf.ln(5)
    
    # KPI table
    pdf.set_font('Helvetica', 'B', 10)
    headers = ['KPI', 'Nilai', 'Delta', 'Status']
    col_widths = [50, 30, 30, 30]
    for i, h in enumerate(headers):
        pdf.cell(col_widths[i], 7, h, 1, 0, 'C')
    pdf.ln()
    
    pdf.set_font('Helvetica', '', 9)
    for kpi in report_data['kpis']:
        pdf.cell(50, 6, kpi['label'], 1)
        pdf.cell(30, 6, str(kpi['value']), 1, 0, 'R')
        pdf.cell(30, 6, kpi['delta'], 1, 0, 'R')
        pdf.cell(30, 6, kpi['status'], 1, 0, 'C')
        pdf.ln()
    
    # Charts sebagai image (dari matplotlib)
    if report_data.get('chart_images'):
        for chart in report_data['chart_images']:
            pdf.add_page()
            pdf.image(chart['path'], x=10, w=190)
    
    buf = io.BytesIO()
    pdf.output(buf)
    buf.seek(0)
    return buf.read()
```

## 7. TROUBLESHOOTING REPLIT

### Problem: WeasyPrint import error

```
Error: "OSError: cannot load library 'pango-1.0'"

Solusi:
1. Pastikan replit.nix sudah benar (lihat Section 1C)
2. Klik "Run" ulang setelah edit replit.nix (Nix rebuild)
3. Jika tetap gagal → gunakan Opsi A (HTML print) atau Opsi B (fpdf2)
```

### Problem: Port sudah dipakai

```
Error: "Address already in use :8000"

Solusi:
Di Replit shell:
  kill $(lsof -t -i:8000) 2>/dev/null
  kill $(lsof -t -i:5173) 2>/dev/null
Lalu Run ulang.
```

### Problem: Memory limit

```
Error: "Killed" atau "MemoryError" saat proses upload besar

Solusi:
1. Tambahkan DUCKDB_MEMORY_LIMIT di config.py (lihat Section 5A)
2. Kurangi MAX_UPLOAD_MB
3. Proses data dalam batch (chunked processing sudah ada di Prompt 2)
4. Upgrade ke Replit paid tier
```

### Problem: Frontend HMR tidak jalan

```
Gejala: Edit React → tidak auto-refresh

Solusi:
Pastikan vite.config.js punya hmr config (lihat Section 3A):
  hmr: { clientPort: 443, protocol: 'wss' }
```

### Problem: Database hilang setelah Repl restart

```
Gejala: Data kosong setelah beberapa hari idle

Penyebab: Replit free tier bisa "sleep" dan kadang
          file hilang jika storage penuh.

Solusi:
1. Backup otomatis sudah ada di Prompt 0 (auto-backup sebelum import)
2. Tambahkan endpoint manual backup:
   POST /api/backup/export → download .duckdb file
3. Untuk production: simpan backup di luar Replit
```

### Problem: Replit "Run" button memulai proses yang salah

```
Pastikan .replit file punya: run = "bash start.sh"
Pastikan start.sh punya: chmod +x start.sh
```
