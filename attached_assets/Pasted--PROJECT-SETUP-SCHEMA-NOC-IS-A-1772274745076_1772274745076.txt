## 2. MASTER HIERARKI ORGANISASI

### Konteks Masalah:

    Data NOC punya inkonsistensi penamaan antar sumber:
    - Site master: "REGIONAL3" (uppercase, tanpa prefix R)
    - Data tiket:  "R03_Inner Jabotabek" (prefix R, underscore, nama)
    - Area TIDAK ADA di site master — hanya ada di data tiket
    
    Solusi: setiap tabel master punya kolom ALIAS untuk mapping otomatis.

### 2A. `master_area`

    CREATE TABLE IF NOT EXISTS master_area (

### 2E. View: `v_hierarchy` (denormalized, untuk kemudahan query)

    CREATE VIEW v_hierarchy AS
    SELECT 
        t.to_id, t.to_name,
        n.nop_id, n.nop_name,
        r.regional_id, r.regional_name,
        a.area_id, a.area_name
    FROM master_to t
    JOIN master_nop n ON t.nop_id = n.nop_id
    JOIN master_regional r ON n.regional_id = r.regional_id
    JOIN master_area a ON r.area_id = a.area_id
    WHERE t.status = 'ACTIVE'
      AND n.status = 'ACTIVE'
      AND r.status = 'ACTIVE'
      AND a.status = 'ACTIVE';

### 2F. `orphan_log` (untuk tracking data yang tidak ter-mapping)

    CREATE TABLE IF NOT EXISTS orphan_log (
        id INTEGER PRIMARY KEY,
        source VARCHAR NOT NULL,           -- 'site_master' / 'ticket'
        level VARCHAR NOT NULL,            -- 'area' / 'regional' / 'nop' / 'to'
        value VARCHAR NOT NULL,            -- nilai yang tidak ter-mapping
        suggested_match VARCHAR,           -- best guess (fuzzy match)
        resolved BOOLEAN DEFAULT FALSE,
        resolved_to VARCHAR,               -- mapping yang dipilih user
        first_seen TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        resolved_at TIMESTAMP,
        ticket_count INTEGER DEFAULT 1     -- berapa tiket terdampak
    );

### 3B. Enrichment Rules (Python, di `backend/services/enrichment_service.py`)

Buat service yang menghitung kolom derived berdasarkan `site_class` dan `site_flag`.Rules ini HARUS persis seperti berikut — ini materialisasi formula Excel:

    def enrich_site(site_class: str, site_flag: str) -> dict:
        """
        Hitung semua kolom derived berdasarkan class × flag.
        Return dict dengan semua kolom derived.
        """
        cls = site_class   # Diamond/Platinum/Gold/Silver/Bronze
        flag = site_flag   # Site Reguler/3T/USO/MP/Femto/No BTS
    
        # 1. site_category
        if flag == "No BTS":
            site_category = "Non-Komersial"
        elif flag == "Site Reguler":
            site_category = "Komersial"
        else:
            site_category = "Non-Komersial"  # 3T, USO/MP, Femto
    
        # 2. site_sub_class
        if flag == "No BTS":
            site_sub_class = "No BTS"
        elif flag == "Site Reguler":
            site_sub_class = cls  # "Diamond", "Gold", dst
        elif flag == "3T":
            site_sub_class = f"{cls}-3T"  # "Bronze-3T", "Gold-3T"
        elif flag == "USO/MP":
            site_sub_class = f"{cls}-USO"
        elif flag == "Femto":
            site_sub_class = f"{cls}-Femto"
        else:
            site_sub_class = cls
    
        # 3. est_technology
        if flag == "No BTS":
            est_technology = "Tidak Ada BTS"
        elif cls in ("Diamond", "Platinum"):
            est_technology = "Multi (2G/3G/4G/5G)"
        elif cls in ("Gold", "Silver"):
            est_technology = "Multi (2G/3G/4G)"
        elif cls == "Bronze" and flag == "Site Reguler":
            est_technology = "Limited (4G / 2G+4G)"
        elif cls == "Bronze" and flag in ("3T", "USO/MP"):
            est_technology = "Single (4G LTE)"
        elif cls == "Bronze" and flag == "Femto":
            est_technology = "Single (4G Femto)"
        else:
            est_technology = "N/A"
    
        # 4. est_transmission
        if flag == "No BTS":
            est_transmission = "Tidak Ada"
        elif cls in ("Diamond", "Platinum"):
            est_transmission = "Fiber Optic + Microwave"
        elif cls in ("Gold", "Silver"):
            est_transmission = "Fiber Optic atau Microwave"
        elif cls == "Bronze" and flag == "Site Reguler":
            est_transmission = "Microwave"
        elif cls == "Bronze" and flag in ("3T", "USO/MP"):
            est_transmission = "VSAT atau Microwave"
        elif cls == "Bronze" and flag == "Femto":
            est_transmission = "WiFi Backhaul / VSAT"
        else:
            est_transmission = "N/A"
    
        # 5. est_power
        if flag == "No BTS":
            est_power = "Tidak Ada"
        elif cls in ("Diamond", "Platinum"):
            est_power = "PLN + Genset + Baterai (redundan)"
        elif cls in ("Gold", "Silver"):
            est_power = "PLN + Baterai"
        elif cls == "Bronze" and flag == "Site Reguler":
            est_power = "PLN + Baterai (minimal)"
        elif cls == "Bronze" and flag in ("3T", "USO/MP"):
            est_power = "Solar Panel + Baterai"
        elif cls == "Bronze" and flag == "Femto":
            est_power = "PLN (rumah/gedung)"
        else:
            est_power = "N/A"
    
        # 6. est_sector
        if flag == "No BTS":
            est_sector = "Tidak Ada"
        elif cls in ("Diamond", "Platinum"):
            est_sector = "3+ Sektor (directional)"
        elif cls in ("Gold", "Silver"):
            est_sector = "3 Sektor"
        elif cls == "Bronze" and flag == "Site Reguler":
            est_sector = "1-2 Sektor"
        elif cls == "Bronze" and flag in ("3T", "USO/MP", "Femto"):
            est_sector = "Omni (360°)"
        else:
            est_sector = "N/A"
    
        # 7. complexity_level (0-5)
        if flag == "No BTS":
            complexity_level = 0
        elif cls in ("Diamond", "Platinum"):
            complexity_level = 5
        elif cls in ("Gold", "Silver"):
            complexity_level = 4
        elif cls == "Bronze" and flag == "Site Reguler":
            complexity_level = 3
        elif cls == "Bronze" and flag in ("3T", "USO/MP"):
            complexity_level = 2
        elif cls == "Bronze" and flag == "Femto":
            complexity_level = 1
        else:
            complexity_level = 0
    
        # 8. est_opex_level
        opex_map = {5: "Very High", 4: "High", 3: "Medium", 2: "Low", 1: "Very Low", 0: "N/A"}
        est_opex_level = opex_map.get(complexity_level, "N/A")
    
        # 9. upgrade_potential
        if cls in ("Diamond", "Platinum"):
            upgrade_potential = "N/A (Tertinggi)"
        elif cls == "Gold":
            upgrade_potential = "Low"
        elif cls == "Silver":
            upgrade_potential = "Medium"
        elif cls == "Bronze":
            upgrade_potential = "High"
        else:
            upgrade_potential = "N/A"
    
        # 10. strategy_focus
        if flag == "No BTS":
            strategy_focus = "Non-Applicable"
        elif cls in ("Diamond", "Platinum"):
            strategy_focus = "Capacity & Quality Management"
        elif cls in ("Gold", "Silver"):
            strategy_focus = "Reliability & Optimization"
        elif cls == "Bronze" and flag == "Site Reguler":
            strategy_focus = "OPEX Efficiency"
        elif cls == "Bronze" and flag in ("3T", "USO/MP"):
            strategy_focus = "Availability & Access"
        elif cls == "Bronze" and flag == "Femto":
            strategy_focus = "Monitoring Minimal"
        else:
            strategy_focus = "N/A"
    
        return {
            "site_category": site_category,
            "site_sub_class": site_sub_class,
            "est_technology": est_technology,
            "est_transmission": est_transmission,
            "est_power": est_power,
            "est_sector": est_sector,
            "complexity_level": complexity_level,
            "est_opex_level": est_opex_level,
            "upgrade_potential": upgrade_potential,
            "strategy_focus": strategy_focus,
        }

## 7. API ENDPOINTS

### 7A. Schema Management (`backend/routers/schema.py`)

    POST /api/schema/init
      → Jalankan semua CREATE TABLE IF NOT EXISTS
      → Jalankan semua INSERT seed data (dengan ON CONFLICT DO NOTHING)
      → Return: {tables_created: [...], seed_data: {...}, status: "success"}
    
    GET /api/schema/status
      → Cek setiap tabel: ada atau tidak, jumlah baris
      → Return: {
          tables: {
            master_area: {exists: true, rows: 4},
            master_regional: {exists: true, rows: 0},
            ...
            noc_tickets: {exists: true, rows: 0},
          },
          total_tables: 14,
          initialized: true  // true jika semua tabel ada
        }
    
    POST /api/schema/seed-reset
      → Reset seed data (master_threshold, master_sla_target) ke default
      → TIDAK menghapus data user (tiket, site, dll)
      → Return: {reset: ["master_threshold", "master_sla_target"]}

### 7B. Threshold API (`backend/routers/threshold.py`)

    GET  /api/threshold
      → Return semua threshold, grouped by category
      → {categories: {MTTR: [...], ESCALATION: [...], COST: [...]}}
    
    GET  /api/threshold/{param_key}
      → Return 1 threshold
    
    PUT  /api/threshold/{param_key}
      → Body: {param_value: 300}
      → Update 1 threshold
      → Return: updated threshold object

## CATATAN PENTING

1. **DuckDB tidak support FOREIGN KEY enforcement** — FK ditulis sebagai dokumentasi, validasi dilakukan di application layer (Python)
2. **Semua seed data pakai ON CONFLICT DO NOTHING** — supaya tidak error jika dijalankan berkali-kali
3. **Enrichment rules di `enrichment_service.py`** belum dipanggil di prompt ini — hanya definisi. Akan dipanggil di Prompt 2 (Upload Engine) dan Prompt 4 (Site Enrichment)
4. **Summary tables kosong** — akan diisi setelah upload data di Prompt 2
5. **v_hierarchy view akan kosong** sampai master_regional/nop/to diisi (Prompt 3 atau upload site master di Prompt 2)
