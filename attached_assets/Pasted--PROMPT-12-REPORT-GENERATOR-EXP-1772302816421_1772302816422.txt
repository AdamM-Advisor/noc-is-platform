## 1. REPORT GENERATOR — KONSEP

    Platform menghasilkan laporan terstruktur secara periodik.
    Setiap level pelaporan memiliki template, cakupan, dan audiens berbeda.
    
    PRINSIP:
      1. TEMPLATE-DRIVEN: Setiap tipe laporan punya template HTML
      2. DATA-FILLED: Template diisi otomatis dari Profiler Engine + Summary
      3. NARASI OTOMATIS: Rule engine (Layer 1-2) + opsional AI (Layer 3-4)
      4. CHART SERVER-SIDE: Chart di-render sebagai gambar untuk PDF
      5. AUDITABLE: Setiap laporan tersimpan di report_history
    
    2 MODE GENERASI:
      Manual (On-Demand): User pilih tipe + scope + periode → Generate
      Scheduled (v2): Cron job → auto-generate pada jadwal tertentu
    
      v1 (prompt ini): Manual only, dengan tombol "Generate Now"
      v2 (future): Scheduled via backend cron/task queue

## 2. DATABASE SCHEMA

    CREATE TABLE report_history (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
    
        -- Report identity
        report_type VARCHAR(20) NOT NULL,  -- 'daily'|'weekly'|'monthly'|'quarterly'|'annual'
        period_start DATE NOT NULL,
        period_end DATE NOT NULL,
        period_label VARCHAR(100),         -- "Juli 2025" | "Minggu 28 (7-13 Jul)" | etc
    
        -- Scope
        entity_level VARCHAR(20) NOT NULL,
        entity_id VARCHAR(50) NOT NULL,
        entity_name VARCHAR(200),
    
        -- Generation info
        generated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        generated_by VARCHAR(100) DEFAULT 'manual',  -- 'manual' | 'scheduled' | username
        generation_time_ms INTEGER,        -- berapa lama generate (ms)
        ai_enhanced BOOLEAN DEFAULT FALSE, -- apakah pakai AI narrative
    
        -- Output files
        pdf_path VARCHAR(500),
        pdf_size_kb INTEGER,
        excel_path VARCHAR(500),
        excel_size_kb INTEGER,
    
        -- Status
        status VARCHAR(20) DEFAULT 'pending',  -- 'pending'|'generating'|'completed'|'failed'
        error_message TEXT,
    
        -- KPI Snapshot (audit trail)
        metadata JSON                      -- snapshot KPI + summary saat generate
    );
    
    CREATE INDEX idx_report_history_type ON report_history(report_type, entity_level, entity_id);
    CREATE INDEX idx_report_history_date ON report_history(generated_at DESC);
    CREATE INDEX idx_report_history_period ON report_history(period_start, period_end);

## 5. PDF GENERATION ENGINE

### 5A. Architecture

    # File: backend/services/report_service.py
    
    class ReportGenerator:
        """
        Master report generator.
        Flow: collect data → render HTML → convert to PDF → save.
        """
    
        def generate(self, report_type, entity_level, entity_id,
                     period_start, period_end, options) -> dict:
            """
            Main entry point.
            """
            # Step 0: Create report_history record (status=pending)
            report_id = self._create_history(report_type, entity_level, 
                                              entity_id, period_start, period_end)
            self._update_status(report_id, 'generating')
    
            try:
                start_time = time.time()
    
                # Step 1: Collect all data needed for this report type
                data = self._collect_data(report_type, entity_level, entity_id,
                                           period_start, period_end)
    
                # Step 2: Generate narratives (rule-based)
                narratives = self._generate_narratives(report_type, data)
    
                # Step 2b: AI-enhanced narrative (optional)
                if options.get('ai_enhanced'):
                    narratives = self._enhance_with_ai(narratives, data)
    
                # Step 3: Render charts as images (for PDF embedding)
                charts = self._render_charts(report_type, data)
    
                # Step 4: Render HTML template
                html = self._render_template(report_type, data, narratives, charts)
    
                # Step 5: Convert HTML → PDF
                pdf_bytes = self._html_to_pdf(html)
                pdf_path = self._save_pdf(report_id, pdf_bytes)
    
                # Step 6: Generate Excel (if applicable)
                excel_path = None
                if options.get('include_excel') and report_type in ('monthly', 'quarterly', 'annual'):
                    excel_bytes = self._generate_excel(report_type, data)
                    excel_path = self._save_excel(report_id, excel_bytes)
    
                # Step 7: Update history
                elapsed = int((time.time() - start_time) * 1000)
                self._update_completed(report_id, pdf_path, excel_path, elapsed, data)
    
                return {
                    "report_id": report_id,
                    "status": "completed",
                    "pdf_path": pdf_path,
                    "excel_path": excel_path,
                    "generation_time_ms": elapsed,
                }
    
            except Exception as e:
                self._update_status(report_id, 'failed', str(e))
                raise

### 5C. Chart Rendering for PDF

    # Charts harus di-render sebagai gambar (PNG) untuk embed di PDF.
    # Pendekatan: gunakan matplotlib untuk server-side chart rendering.
    
    import matplotlib
    matplotlib.use('Agg')  # non-interactive backend
    import matplotlib.pyplot as plt
    import io, base64
    
    class ChartRenderer:
        """Render charts as base64 PNG images for PDF embedding."""
    
        def render_trend_line(self, data, title, target=None) -> str:
            """Render trend line chart, return base64 PNG."""
            fig, ax = plt.subplots(figsize=(8, 3))
    
            periods = [d['period'] for d in data]
            values = [d['value'] for d in data]
    
            ax.plot(periods, values, 'o-', color='#1e40af', linewidth=2, markersize=6)
    
            if target:
                ax.axhline(y=target, color='#DC2626', linestyle='--', label=f'Target {target}%')
    
            ax.set_title(title, fontsize=11, fontweight='bold')
            ax.grid(True, alpha=0.3)
            ax.legend()
    
            plt.tight_layout()
            return self._fig_to_base64(fig)
    
        def render_heatmap(self, cells, y_labels, x_labels, title) -> str:
            """Render heatmap as PNG."""
            fig, ax = plt.subplots(figsize=(10, 4))
            im = ax.imshow(cells, cmap='YlOrRd', aspect='auto')
    
            ax.set_xticks(range(len(x_labels)))
            ax.set_xticklabels(x_labels, fontsize=7)
            ax.set_yticks(range(len(y_labels)))
            ax.set_yticklabels(y_labels, fontsize=8)
            ax.set_title(title, fontsize=11, fontweight='bold')
    
            fig.colorbar(im, ax=ax, shrink=0.8)
            plt.tight_layout()
            return self._fig_to_base64(fig)
    
        def render_pareto(self, labels, values, title) -> str:
            """Render Pareto bar + cumulative line."""
            fig, ax1 = plt.subplots(figsize=(8, 4))
    
            sorted_data = sorted(zip(labels, values), key=lambda x: -x[1])
            labels_s = [d[0] for d in sorted_data]
            values_s = [d[1] for d in sorted_data]
            cumulative = [sum(values_s[:i+1])/sum(values_s)*100 for i in range(len(values_s))]
    
            ax1.bar(labels_s, values_s, color='#3B82F6', alpha=0.8)
            ax1.set_ylabel('Volume')
    
            ax2 = ax1.twinx()
            ax2.plot(labels_s, cumulative, 'o-', color='#DC2626', linewidth=2)
            ax2.axhline(y=80, color='#DC2626', linestyle=':', alpha=0.5)
            ax2.set_ylabel('Kumulatif %')
    
            ax1.set_title(title, fontsize=11, fontweight='bold')
            plt.tight_layout()
            return self._fig_to_base64(fig)
    
        def render_radar(self, labels, values, title) -> str:
            """Render radar chart for risk components or KPI comparison."""
            angles = [n / float(len(labels)) * 2 * 3.14159 for n in range(len(labels))]
            angles += angles[:1]
            values_plot = values + values[:1]
    
            fig, ax = plt.subplots(figsize=(5, 5), subplot_kw=dict(polar=True))
            ax.plot(angles, values_plot, 'o-', color='#1e40af', linewidth=2)
            ax.fill(angles, values_plot, alpha=0.15, color='#1e40af')
            ax.set_xticks(angles[:-1])
            ax.set_xticklabels(labels, fontsize=8)
            ax.set_title(title, fontsize=11, fontweight='bold', y=1.1)
    
            plt.tight_layout()
            return self._fig_to_base64(fig)
    
        def _fig_to_base64(self, fig) -> str:
            buf = io.BytesIO()
            fig.savefig(buf, format='png', dpi=150, bbox_inches='tight',
                        facecolor='white', edgecolor='none')
            plt.close(fig)
            buf.seek(0)
            return base64.b64encode(buf.read()).decode('utf-8')

    import httpx
    
    class AINarrativeService:
        """
        Opsional: enhance rule-based narratives with AI.
        Toggle ON/OFF di UI. Fallback ke rule-based jika gagal.
    
        KEAMANAN: AI HANYA menerima aggregated data, BUKAN raw tickets.
        """
    
        API_URL = "https://api.anthropic.com/v1/messages"
        MODEL = "claude-sonnet-4-20250514"
    
        def enhance_narrative(self, report_type: str, structured_data: dict) -> str:
            """
            Kirim structured insights ke Claude API.
            Return narasi yang lebih natural dan insightful.
            """
            # Sanitize: hanya kirim aggregated data
            safe_data = self._sanitize(structured_data)
    
            prompt = self._build_prompt(report_type, safe_data)
    
            try:
                response = httpx.post(
                    self.API_URL,
                    headers={"Content-Type": "application/json"},
                    json={
                        "model": self.MODEL,
                        "max_tokens": 2000,
                        "messages": [{"role": "user", "content": prompt}]
                    },
                    timeout=15.0
                )
    
                if response.status_code == 200:
                    data = response.json()
                    return data['content'][0]['text']
                else:
                    return None  # fallback to rule-based
    
            except Exception:
                return None  # fallback to rule-based
    
        def _sanitize(self, data: dict) -> dict:
            """
            Hapus data sensitif. AI TIDAK BOLEH menerima:
            - Raw ticket data / ticket numbers
            - PIC names / emails
            - NOSSA numbers
            """
            safe = {
                'entity_name': data.get('entity_name'),
                'entity_level': data.get('entity_level'),
                'period': data.get('period'),
                'kpis': data.get('kpis'),           # aggregated values only
                'trends': data.get('trends'),         # directions + slopes
                'rankings': data.get('rankings'),     # percentiles
                'recommendations': data.get('recommendations'),  # rule engine output
                'risk_summary': data.get('risk_summary'),

## CATATAN PENTING

1. **WeasyPrint > reportlab**: WeasyPrint meng-convert HTML → PDF.Artinya template HTML yang sama bisa dipakai untuk preview (browser)dan PDF (download). Satu template untuk 2 output.
  
2. **Chart = matplotlib server-side**: Recharts (frontend) tidak bisadi-render di backend. Gunakan matplotlib untuk generate PNGyang di-embed sebagai base64 ke HTML → PDF.
  
3. **Excel = lampiran, bukan primary**: PDF selalu primary output.Excel hanya untuk bulanan+ sebagai data pendukung.
  
4. **AI-Enhanced = opsional**: Default OFF. Butuh API key.Jika gagal, fallback otomatis ke rule-based. User diberi notif.
  
5. **Report History = audit trail**: Setiap laporan tersimpan dengansnapshot KPI. Bisa di-download ulang kapan saja.
  
6. **Performance**: Monthly report (13 Regional) = ~3-5 detik each.Batch 151 TO daily = ~10-15 menit total. v2 pakai task queue.
  
7. **Light mode**: Semua template HTML pakai background putih,warna accent #1E40AF (biru), status sesuai traffic light.
