## 1. KONSEP & ANALOGI DTC

```
ADOPSI DTC OTOMOTIF KE NOC:

  DTC Otomotif              NDC NOC-IS
  ─────────────             ────────────
  DTC Code (P0301)       →  NDC Code (NDC-PWR-0001)
  ECU Sensor Readings    →  Alarm Snapshot: severity, NE class, RAT, impact
  Freeze Frame           →  Alarm Snapshot: co-occurring alarms + temporal
  Symptom List           →  Symptom Patterns: primer, sekunder, negatif
  Diagnostic Steps       →  Diagnostic Tree: decision tree + branching
  Repair Guide           →  SOP Penanganan: multi-path + eskalasi + preventif
  Severity Level         →  Priority Score (HIGH/MEDIUM/LOW)
  
POSISI NDC DALAM ARSITEKTUR:

  Data Tiket (noc_tickets)
       │
       │ aggregate per (rc_category, rc_1, rc_2)
       ▼
  ┌─────────────────┐
  │  NDC GENERATOR  │ ← auto-discover dari data historis
  │                 │ ← generate 4 tab enrichment per NDC
  │                 │ ← bangun confusion matrix INAP vs Engineer
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │  NDC KNOWLEDGE  │ ← tabel ndc_entries + 6 tabel enrichment
  │  BASE           │ ← 4 tab per entry: Snapshot/Symptoms/Diag/SOP
  │                 │ ← browsable, searchable, curation, exportable
  └────────┬────────┘
           │
           ├──► Profiler: NDC distribution per site/entity
           ├──► Report Card: Top 3 NDC per entity
           ├──► Rekomendasi: R21-R23 berbasis NDC
           ├──► Laporan: Section Top NDC di bulanan+
           └──► Future: Site DNA L1 Base

PERBEDAAN DENGAN PROFILER GANGGUAN (Prompt 8):
  Prompt 8: Pareto gangguan, distribusi fault level, RC analysis
            → DESKRIPTIF: "apa yang terjadi"
  NDC:      Knowledge base terstruktur dengan kode unik per pattern
            → KODIFIKASI: "ensiklopedia gangguan" yang reusable & learnable
            → Setiap entry = mini-handbook lengkap dengan SOP
```

**Auto-generation logic — Alarm Snapshot:**

```python
def generate_alarm_snapshot(self, ndc_code: str, rc_category: str, rc_1: str, rc_2: str):
    """
    100% otomatis. Aggregate noc_tickets per NDC.
    """
    
    # === Typical Profile ===
    # Per field: hitung distribusi, ambil top 2-3 values sebagai string
    sql_profile = """
    WITH ndc_tickets AS (
        SELECT t.*, ms.site_class, ms.flag_3t, ms.flag_kriteria
        FROM noc_tickets t
        LEFT JOIN master_site ms ON t.site_id = ms.site_id
        WHERE t.rc_category = ? AND t.rc_1 = ?
          AND (? IS NULL OR t.rc_2 = ?)
    )
    SELECT
        -- Severity: top values + pct
        (SELECT STRING_AGG(sv || ' (' || pct || '%)', ', ')
         FROM (
            SELECT severity as sv, ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets),1) as pct
            FROM ndc_tickets GROUP BY severity ORDER BY COUNT(*) DESC LIMIT 3
         )) as typical_severity,
        
        -- NE class: top values
        (SELECT STRING_AGG(nc || ' (' || pct || '%)', ', ')
         FROM (
            SELECT ne_class as nc, ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets),1) as pct
            FROM ndc_tickets WHERE ne_class IS NOT NULL GROUP BY ne_class ORDER BY COUNT(*) DESC LIMIT 3
         )) as typical_ne_class,
        
        -- Fault level
        (SELECT STRING_AGG(fl || ' (' || pct || '%)', ', ')
         FROM (
            SELECT fault_level as fl, ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets),1) as pct
            FROM ndc_tickets WHERE fault_level IS NOT NULL GROUP BY fault_level ORDER BY COUNT(*) DESC LIMIT 2
         )) as typical_fault_level,
        
        -- Impact
        (SELECT STRING_AGG(imp || ' (' || pct || '%)', ', ')
         FROM (
            SELECT impact as imp, ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets),1) as pct
            FROM ndc_tickets WHERE impact IS NOT NULL GROUP BY impact ORDER BY COUNT(*) DESC LIMIT 2
         )) as typical_impact,
        
        -- Type ticket
        (SELECT STRING_AGG(tt || ' (' || pct || '%)', ', ')
         FROM (
            SELECT type_ticket as tt, ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets),1) as pct
            FROM ndc_tickets GROUP BY type_ticket ORDER BY COUNT(*) DESC LIMIT 2
         )) as typical_type_ticket,
        
        -- RAT
        (SELECT STRING_AGG(r || ' (' || pct || '%)', ', ')
         FROM (
            SELECT rat as r, ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets),1) as pct
            FROM ndc_tickets WHERE rat IS NOT NULL GROUP BY rat ORDER BY COUNT(*) DESC LIMIT 3
         )) as typical_rat,
        
        COUNT(*) as sample_size
        
    FROM ndc_tickets
    """
    
    # === Temporal Signature ===
    sql_temporal = """
    WITH ndc_tickets AS (
        SELECT * FROM noc_tickets
        WHERE rc_category = ? AND rc_1 = ? AND (? IS NULL OR rc_2 = ?)
    ),
    hourly AS (
        SELECT calc_hour_of_day as h, COUNT(*) as cnt,
               ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets), 1) as pct
        FROM ndc_tickets GROUP BY calc_hour_of_day
    ),
    -- Cari consecutive hour range dengan volume tertinggi (3-hour window)
    peak_range AS (
        SELECT h as start_h, 
               SUM(pct) OVER (ORDER BY h ROWS BETWEEN CURRENT ROW AND 2 FOLLOWING) as range_pct
        FROM hourly ORDER BY range_pct DESC LIMIT 1
    )
    SELECT 
        -- Peak hours range
        (SELECT LPAD(start_h::TEXT, 2, '0') || ':00 — ' || LPAD(((start_h+5)%24)::TEXT, 2, '0') || ':00 (' || ROUND(range_pct,0) || '% tiket)' FROM peak_range) as peak_hours_range,
        
        -- Peak days (top 2)
        (SELECT STRING_AGG(day_name || ' (' || pct || '%)', ', ')
         FROM (
            SELECT CASE calc_day_of_week 
                WHEN 0 THEN 'Minggu' WHEN 1 THEN 'Senin' WHEN 2 THEN 'Selasa'
                WHEN 3 THEN 'Rabu' WHEN 4 THEN 'Kamis' WHEN 5 THEN 'Jumat' WHEN 6 THEN 'Sabtu'
            END as day_name,
            ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets), 1) as pct
            FROM ndc_tickets GROUP BY calc_day_of_week ORDER BY COUNT(*) DESC LIMIT 2
         )) as peak_days,
        
        -- Seasonal: bulan dengan volume > 120% rata-rata
        (SELECT STRING_AGG(month_name, ', ')
         FROM (
            SELECT CASE calc_month 
                WHEN 1 THEN 'Jan' WHEN 2 THEN 'Feb' WHEN 3 THEN 'Mar' WHEN 4 THEN 'Apr'
                WHEN 5 THEN 'Mei' WHEN 6 THEN 'Jun' WHEN 7 THEN 'Jul' WHEN 8 THEN 'Agu'
                WHEN 9 THEN 'Sep' WHEN 10 THEN 'Okt' WHEN 11 THEN 'Nov' WHEN 12 THEN 'Des'
            END as month_name, COUNT(*) as cnt
            FROM ndc_tickets GROUP BY calc_month
            HAVING COUNT(*) > 1.2 * (SELECT COUNT(*)/12.0 FROM ndc_tickets)
            ORDER BY calc_month
         )) as seasonal_months
    FROM ndc_tickets LIMIT 1
    """
    
    # === Site Signature ===
    sql_site = """
    WITH ndc_tickets AS (
        SELECT t.*, ms.site_class, ms.flag_3t, ms.flag_kriteria,
               mr.regional_id, mr.name as regional_name
        FROM noc_tickets t
        LEFT JOIN master_site ms ON t.site_id = ms.site_id
        LEFT JOIN master_to mt ON t.calc_to_id = mt.to_id
        LEFT JOIN master_nop mn ON mt.nop_id = mn.nop_id
        LEFT JOIN master_regional mr ON mn.regional_id = mr.regional_id
        WHERE t.rc_category = ? AND t.rc_1 = ? AND (? IS NULL OR t.rc_2 = ?)
    )
    SELECT 
        -- Site class distribution
        (SELECT STRING_AGG(sc || ' (' || pct || '%)', ', ')
         FROM (
            SELECT site_class as sc, ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets),1) as pct
            FROM ndc_tickets WHERE site_class IS NOT NULL GROUP BY site_class ORDER BY COUNT(*) DESC
         )) as site_class_distribution,
        
        -- 3T percentage
        ROUND(AVG(CASE WHEN flag_3t THEN 1 ELSE 0 END) * 100, 1) as pct_3t,
        
        -- Kriteria percentage
        ROUND(AVG(CASE WHEN flag_kriteria THEN 1 ELSE 0 END) * 100, 1) as pct_kriteria,
        
        -- Top regions
        (SELECT STRING_AGG(rn || ' (' || pct || '%)', ', ')
         FROM (
            SELECT regional_name as rn, ROUND(COUNT(*)*100.0/(SELECT COUNT(*) FROM ndc_tickets),1) as pct
            FROM ndc_tickets WHERE regional_name IS NOT NULL GROUP BY regional_name ORDER BY COUNT(*) DESC LIMIT 3
         )) as top_regions
         
    FROM ndc_tickets
    """
    
    # === Co-occurring Alarms ===
    sql_co_alarms = """
    WITH ndc_base AS (
        SELECT ticket_number_inap, site_id, occured_time, rc_1
        FROM noc_tickets
        WHERE rc_category = ? AND rc_1 = ? AND (? IS NULL OR rc_2 = ?)
    ),
    co_tickets AS (
        SELECT 
            t2.rc_category as co_rc_cat,
            t2.rc_1 as co_rc_1,
            COALESCE(t2.summary, t2.rc_1) as co_description,
            EXTRACT(EPOCH FROM t2.occured_time - t1.occured_time) / 60.0 as lag_min
        FROM ndc_base t1
        JOIN noc_tickets t2 
            ON t1.site_id = t2.site_id
            AND t2.ticket_number_inap != t1.ticket_number_inap
            AND ABS(EXTRACT(EPOCH FROM t2.occured_time - t1.occured_time)) < 14400  -- ±4 jam
        WHERE t2.rc_1 != ?  -- exclude NDC yang sama
    )
    SELECT 
        co_rc_1 as co_alarm_description,
        co_rc_cat as co_alarm_rc_category,
        co_rc_1 as co_alarm_rc_1,
        ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM ndc_base), 1) as co_occurrence_pct,
        ROUND(AVG(lag_min), 0) as typical_lag_min,
        CASE WHEN AVG(lag_min) > 0 THEN 'after' 
             WHEN AVG(lag_min) < 0 THEN 'before' 
             ELSE 'concurrent' END as lag_direction,
        COUNT(*) as sample_size
    FROM co_tickets
    GROUP BY co_rc_cat, co_rc_1, co_description
    HAVING COUNT(*) >= 10
    ORDER BY COUNT(*) DESC
    LIMIT 5
    """
    
    # Execute semua, insert ke ndc_alarm_snapshot + ndc_co_occurring_alarms
```

**Auto-extraction logic — Symptoms:**

```python
def extract_symptoms(self, ndc_code: str, rc_category: str, rc_1: str, rc_2: str):
    """
    Extract symptom patterns dari summary/description.
    Pendekatan: keyword frequency matching (bukan NLP).
    ~70% auto, ~30% perlu kurasi (terutama negative symptoms).
    """
    
    # Step 1: Ambil semua summary + description untuk NDC ini
    sql = """
    SELECT summary, description 
    FROM noc_tickets
    WHERE rc_category = ? AND rc_1 = ?
      AND (? IS NULL OR rc_2 = ?)
      AND (summary IS NOT NULL OR description IS NOT NULL)
    """
    texts = db.execute(sql, [rc_category, rc_1, rc_2, rc_2])
    total = len(texts)
    if total == 0:
        return []
    
    # Step 2: Predefined symptom keyword patterns
    # Domain-specific patterns untuk NOC telekomunikasi
    SYMPTOM_PATTERNS = {
        "Site unreachable / NE down": {
            "patterns": [r"site.*down", r"ne.*down", r"unreachable", r"not reachable", r"site.*off", r"ne.*off"],
            "source": "summary"
        },
        "Multiple alarm: Power fail + Battery low": {
            "patterns": [r"power.*fail.*batt", r"batt.*low.*power", r"power.*batt", r"power fail"],
            "source": "alarm_correlation"
        },
        "EAS tidak muncul / no external alarm": {
            "patterns": [r"eas.*tidak", r"eas.*no", r"no.*eas", r"eas.*off", r"external alarm.*tidak"],
            "source": "description"
        },
        "Coverage gap di area site": {
            "patterns": [r"coverage.*gap", r"no.*coverage", r"blank.*spot", r"layanan.*terganggu"],
            "source": "impact_analysis"
        },
        "User complaint / gangguan layanan": {
            "patterns": [r"user.*complain", r"pelanggan.*keluh", r"complaint", r"gangguan.*layanan", r"customer"],
            "source": "description"
        },
        "Neighboring site overloaded": {
            "patterns": [r"neighbor.*overload", r"traffic.*shift", r"cell.*congestion", r"handover.*fail"],
            "source": "description"
        },
        "Alarm berulang / flapping": {
            "patterns": [r"flapping", r"berulang", r"intermittent", r"recurring", r"repeat"],
            "source": "summary"
        },
        "Timeout / response lambat": {
            "patterns": [r"timeout", r"time.*out", r"lambat", r"slow.*response", r"delay"],
            "source": "description"
        },
        "Link down / transmisi putus": {
            "patterns": [r"link.*down", r"transmisi.*putus", r"fo.*putus", r"fiber.*cut"],
            "source": "summary"
        },
    }
    
    # Step 3: Hitung frekuensi per pattern
    import re
    symptoms = []
    for symptom_name, config in SYMPTOM_PATTERNS.items():
        match_count = 0
        for row in texts:
            combined = f"{row.get('summary','')} {row.get('description','')}".lower()
            if any(re.search(p, combined) for p in config["patterns"]):
                match_count += 1
        
        freq_pct = round(match_count / total * 100, 1)
        
        if freq_pct >= 5:  # minimum threshold
            symptoms.append({
                'ndc_code': ndc_code,
                'symptom_text': symptom_name,
                'symptom_type': 'primary' if freq_pct >= 30 else 'secondary',
                'frequency_pct': freq_pct,
                'confidence': 'high' if freq_pct >= 50 else ('medium' if freq_pct >= 20 else 'low'),
                'source': config["source"],
                'is_auto_generated': True,
            })
    
    symptoms.sort(key=lambda x: -x['frequency_pct'])
    
    # Step 4: Generate negative symptoms dari confusion matrix
    # "Jika alarm X ada, kemungkinan bukan NDC ini → cek NDC lain"
    negatives = self._generate_negative_symptoms(ndc_code, rc_category)
    symptoms.extend(negatives)
    
    return symptoms

def _generate_negative_symptoms(self, ndc_code: str, rc_category: str) -> list:
    """
    Auto-generate negative symptoms dari confusion matrix.
    Logika: jika INAP sering salah klasifikasi dari kategori lain ke kategori ini,
    maka alarm khas kategori lain = negative symptom.
    """
    sql = """
    SELECT inap_rc_category, confirmed_rc_category, match_pct, ticket_count
    FROM ndc_confusion_matrix
    WHERE confirmed_rc_category = ?
      AND inap_rc_category != ?
      AND ticket_count >= 20
    ORDER BY ticket_count DESC
    LIMIT 3
    """
    rows = db.execute(sql, [rc_category, rc_category])
    
    negatives = []
    CATEGORY_ALARM_HINTS = {
        'Transmisi': 'alarm transmisi / link down / FO putus',
        'Radio': 'alarm radio / RF fault / VSWR high',
        'Power': 'alarm power / PLN off / baterai low',
        'Aktivitas': 'alarm aktivitas pihak ketiga / galian',
        'HW': 'alarm hardware / module fault',
    }
    
    for row in rows:
        other_cat = row['inap_rc_category']
        if other_cat in CATEGORY_ALARM_HINTS:
            negatives.append({
                'ndc_code': ndc_code,
                'symptom_text': f"TIDAK ada {CATEGORY_ALARM_HINTS[other_cat]}",
                'symptom_type': 'negative',
                'frequency_pct': None,
                'confidence': 'medium',
                'source': 'confusion_matrix',
                'negative_note': f"Jika ada {CATEGORY_ALARM_HINTS[other_cat]}, kemungkinan bukan {ndc_code} — cek kategori {other_cat}.",
                'is_auto_generated': True,
            })
    
    return negatives
```

### 2D. Tab 3 — Diagnostic Tree (= DTC Diagnostic Steps)

```sql
-- Decision tree per NDC: langkah diagnostik dengan branching.
-- Skeleton auto-generated, detail butuh kurasi manual.

CREATE TABLE ndc_diagnostic_steps (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    ndc_code VARCHAR REFERENCES ndc_entries(ndc_code),
    
    step_number INTEGER NOT NULL,               -- 1, 2, 3, ...
    action TEXT NOT NULL,                       -- "Cek status PLN area"
    expected_result TEXT,                       -- "Konfirmasi PLN off di area site"
```

```python
def generate_diagnostic_skeleton(self, ndc_code: str, rc_category: str, rc_1: str, rc_2: str):
    """
    Generate SKELETON diagnostic tree dari pola resolusi historis.
    ~40% auto, ~60% perlu kurasi (branching logic).
    
    Pendekatan: 
    - Analisis MTTR stages: quartile distribusi resolusi
    - Tiap quartile = 1 "stage" yang menjadi step skeleton
    - Success rate = % tiket yang resolved di MTTR stage tersebut
    """
    
    # Step 1: Hitung MTTR quartiles untuk NDC ini
    sql_quartiles = """
    SELECT 
        PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY calc_restore_time_min) as q1,
        PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY calc_restore_time_min) as q2,
        PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY calc_restore_time_min) as q3,
        PERCENTILE_CONT(0.90) WITHIN GROUP (ORDER BY calc_restore_time_min) as q4,
        COUNT(*) as total
    FROM noc_tickets
    WHERE rc_category = ? AND rc_1 = ? AND (? IS NULL OR rc_2 = ?)
      AND calc_restore_time_min IS NOT NULL
    """
    q = db.execute_one(sql_quartiles, [rc_category, rc_1, rc_2, rc_2])
    total = q['total']
    
    # Step 2: Hitung % resolved per stage
    sql_stages = """
    SELECT 
        CASE 
            WHEN calc_restore_time_min <= ? THEN 'stage_1'
            WHEN calc_restore_time_min <= ? THEN 'stage_2'
            WHEN calc_restore_time_min <= ? THEN 'stage_3'
            ELSE 'stage_4'
        END as stage,
        COUNT(*) as cnt,
        ROUND(COUNT(*) * 100.0 / ?, 1) as pct,
        MODE(resolution_action) as common_action,
        ROUND(AVG(calc_restore_time_min), 0) as avg_duration
    FROM noc_tickets
    WHERE rc_category = ? AND rc_1 = ? AND (? IS NULL OR rc_2 = ?)
      AND calc_restore_time_min IS NOT NULL
    GROUP BY stage
    ORDER BY stage
    """
    stages = db.execute(sql_stages, [q['q1'], q['q2'], q['q3'], total, rc_category, rc_1, rc_2, rc_2])
    
    # Step 3: Generate skeleton steps
    skeleton = [
        {
            'step_number': 1,
            'action': f"Identifikasi dan konfirmasi gangguan: {rc_1}" + (f" — {rc_2}" if rc_2 else ""),
            'expected_result': f"Konfirmasi bahwa root cause adalah {rc_1}",
            'if_yes': "Lanjut ke Step 2",
            'if_no': "Bukan gangguan ini — evaluasi ulang klasifikasi",
            'avg_duration_min': 5,
            'success_rate_at_step': None,
            'cumulative_resolve_pct': 0,
            'is_auto_generated': True,
        }
    ]
    
    cumulative = 0
    for i, stage in enumerate(stages):
        cumulative += stage['pct']
        skeleton.append({
            'step_number': i + 2,
            'action': stage['common_action'] or f"Resolusi stage {i+1} (MTTR ~{stage['avg_duration']}m)",
            'expected_result': "Gangguan teratasi",
            'if_yes': "Verifikasi dan close tiket" if i == len(stages)-1 else f"Lanjut ke Step {i+3}",
            'if_no': "Eskalasi ke level berikutnya" if i < len(stages)-1 else "Eskalasi ke management",
            'avg_duration_min': stage['avg_duration'],
            'success_rate_at_step': stage['pct'],
            'cumulative_resolve_pct': round(cumulative, 1),
            'is_auto_generated': True,
        })
    
    # Last step: verifikasi
    skeleton.append({
        'step_number': len(skeleton) + 1,
        'action': "Verifikasi: semua NE up, alarm clear, layanan normal",
        'expected_result': "Site fully operational",
        'if_yes': "Close tiket. Update RCA.",
        'if_no': "Ada alarm tersisa — buat tiket baru per alarm type",
        'avg_duration_min': 15,
        'success_rate_at_step': round(100 - cumulative, 1),
        'cumulative_resolve_pct': 100.0,
        'is_auto_generated': True,
    })
    
    return skeleton
```

```python
def generate_resolution_paths(self, ndc_code: str, rc_category: str, rc_1: str, rc_2: str):
    """
    Cluster tiket NDC menjadi resolution paths.
    ~80% auto (clustering + stats), ~20% kurasi (naming + notes).
    """
    
    # Clustering: 5 natural paths
    sql = """
    WITH ndc_tickets AS (
        SELECT * FROM noc_tickets
        WHERE rc_category = ? AND rc_1 = ? AND (? IS NULL OR rc_2 = ?)
          AND calc_restore_time_min IS NOT NULL
    ),
    total AS (SELECT COUNT(*) as cnt FROM ndc_tickets)
    SELECT 
        CASE 
            WHEN is_auto_resolved = 'Auto Resolved' THEN 'Auto-Resolve'
            WHEN escalation_level IS NOT NULL AND escalation_level > 0 THEN 'Eskalasi'
            WHEN calc_restore_time_min <= (SELECT PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY calc_restore_time_min) FROM ndc_tickets) 
                THEN 'Standard Resolution'
            ELSE 'Extended Resolution'
        END as path_cluster,
        COUNT(*) as cnt,
        ROUND(COUNT(*) * 100.0 / (SELECT cnt FROM total), 1) as probability_pct,
        ROUND(AVG(calc_restore_time_min), 0) as avg_mttr_min,
        ROUND(AVG(CASE WHEN calc_is_sla_met THEN 1 ELSE 0 END) * 100, 1) as sla_met_pct,
        MODE(resolution_action) as common_action
    FROM ndc_tickets
    GROUP BY path_cluster
    HAVING COUNT(*) >= 5
    ORDER BY cnt DESC
    """
    
    paths = db.execute(sql, [rc_category, rc_1, rc_2, rc_2])
    
    for i, path in enumerate(paths):
        # Insert path
        path_id = db.insert('ndc_resolution_paths', {

            'path_name': path['path_cluster'],
            'sort_order': i,
```

```python
class NDCDiscoveryEngine:
    """Orchestrator: discovery + semua enrichments."""
    
    CATEGORY_MAP = {
        'Power':      {'code': 'PWR', 'name': 'Power System'},
        'Radio':      {'code': 'RAD', 'name': 'Radio Access'},
        'Transmisi':  {'code': 'TRX', 'name': 'Transmission'},
        'Aktivitas':  {'code': 'AKT', 'name': 'Aktivitas Pihak Ketiga'},
        'CME':        {'code': 'CME', 'name': 'Civil Mechanical Electrical'},
        'LOGIC':      {'code': 'LOG', 'name': 'Logic/Software'},
        'HW':         {'code': 'HW',  'name': 'Hardware'},
    }
    DEFAULT_CATEGORY = {'code': 'OTH', 'name': 'Other'}
    MIN_TICKETS = 10
    
    def full_refresh(self):
        """
        Full refresh: discovery + 4 tab enrichment per NDC.
        Dipanggil setelah upload data baru.
        Existing NDC codes dipertahankan — hanya stats yang di-update.
        NDC baru ditambahkan jika ditemukan kombinasi baru.
        """
        entries = self._discover_entries()
        
        for entry in entries:
            code = entry['ndc_code']
            rc_cat = entry['rc_category']
            rc1 = entry['rc_1']
            rc2 = entry['rc_2']
            
            # Upsert base entry
            self._upsert_entry(entry)
            
            # Tab 1: Alarm Snapshot (100% auto)
            self.generate_alarm_snapshot(code, rc_cat, rc1, rc2)
            
            # Tab 2: Symptoms (70% auto)
            self.extract_symptoms(code, rc_cat, rc1, rc2)
            
            # Tab 3: Diagnostic Tree skeleton (40% auto)
            if not self._has_reviewed_diagnostics(code):
                self.generate_diagnostic_skeleton(code, rc_cat, rc1, rc2)
            
            # Tab 4: Resolution Paths (80% auto)
            self.generate_resolution_paths(code, rc_cat, rc1, rc2)
            self.generate_escalation_matrix(code)
            # Preventive actions: NOT auto-generated (10% — mostly manual)
        
        # Global: Confusion Matrix
        self.build_confusion_matrix()
        
        # Global: Site Distribution
        self.refresh_site_distributions()
    
    def _has_reviewed_diagnostics(self, ndc_code: str) -> bool:
        """Jangan overwrite diagnostic tree yang sudah di-kurasi manual."""
        sql = "SELECT COUNT(*) as cnt FROM ndc_diagnostic_steps WHERE ndc_code = ? AND reviewed = TRUE"
        return db.execute_one(sql, [ndc_code])['cnt'] > 0
    
    def _calc_priority(self, row) -> float:
        """Priority score 0-100: volume(30%) + breach(30%) + severity(20%) + MTTR(20%)"""
        import math
        vol = min(100, math.log10(max(row['total_tickets'], 1)) / math.log10(50000) * 100)
        breach = min(100, row['sla_breach_pct'] * 2)
        sev = min(100, row['pct_critical'] * 2 + row['pct_major'])
        mttr = min(100, (row['avg_mttr_min'] / 1440) * 100)
        return round(vol * 0.3 + breach * 0.3 + sev * 0.2 + mttr * 0.2, 1)
```

## 7. API ENDPOINTS

```
# ── NDC Core ──
GET    /api/ndc                             → list (filter, sort, paginate)
GET    /api/ndc/{code}                      → detail + semua enrichments (4 tab)
POST   /api/ndc/refresh                     → full refresh: discovery + enrichments
PUT    /api/ndc/{code}                      → update kurasi (status, notes, differentiator)
DELETE /api/ndc/{code}                      → soft-delete
POST   /api/ndc/merge                       → merge 2 NDC entries
GET    /api/ndc/export                      → export CSV/Excel

# ── Tab 1: Alarm Snapshot ──
GET    /api/ndc/{code}/alarm-snapshot       → snapshot + co-occurring alarms

# ── Tab 2: Symptoms ──
GET    /api/ndc/{code}/symptoms             → all symptoms (primary/secondary/negative)
POST   /api/ndc/{code}/symptoms             → add symptom (kurasi)
PUT    /api/ndc/{code}/symptoms/{id}        → edit symptom
DELETE /api/ndc/{code}/symptoms/{id}        → delete symptom

# ── Tab 3: Diagnostic Tree ──
GET    /api/ndc/{code}/diagnostic-tree      → all steps with branching
PUT    /api/ndc/{code}/diagnostic-tree      → bulk update steps (kurasi)
POST   /api/ndc/{code}/diagnostic-tree/step → add step
DELETE /api/ndc/{code}/diagnostic-tree/{id} → delete step

# ── Tab 4: SOP ──
GET    /api/ndc/{code}/sop                  → paths + escalation + preventive
PUT    /api/ndc/{code}/sop/paths/{id}       → edit path (name, notes)
PUT    /api/ndc/{code}/sop/paths/{id}/steps → edit steps per path
PUT    /api/ndc/{code}/sop/escalation       → bulk update escalation matrix
POST   /api/ndc/{code}/sop/preventive       → add preventive action
PUT    /api/ndc/{code}/sop/preventive/{id}  → edit preventive action
DELETE /api/ndc/{code}/sop/preventive/{id}  → delete preventive action

# ── Confusion Matrix ──
GET    /api/ndc/confusion-matrix            → matrix INAP vs confirmed

# ── NDC per Entity ──
GET    /api/ndc/site/{site_id}              → NDC distribution per site
GET    /api/ndc/entity/{level}/{id}         → NDC distribution per entity
```
